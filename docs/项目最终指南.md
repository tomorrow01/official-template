## 1. 项目概述

本文档是官网项目的综合指南，整合了技术架构、前端开发、后端开发、管理后台以及API接口等方面的关键信息，为项目的开发、部署和维护提供全面的参考。

### 1.1 项目目标

本官网项目旨在提供一个功能完善、性能优化的企业官方网站，包含轮播图展示、文章管理、案例展示、服务介绍和内容管理等核心功能模块，并通过管理后台实现便捷的内容管理。

### 1.2 项目架构

项目采用前后端分离的架构设计：
- **前端**：基于Nuxt.js + TypeScript + Element Plus构建的响应式网站
- **后端**：基于Node.js + Express + MongoDB构建的RESTful API服务
- **管理后台**：基于Vue 3 + Vite + Element Plus构建的内容管理系统

## 2. 技术栈概览

### 2.1 前端技术栈
- **框架**：Nuxt.js 3.x
- **语言**：TypeScript
- **UI组件**：Element Plus

### 2.3 管理后台技术栈
- **框架**：Vue.js 3.x
- **构建工具**：Vite
- **UI组件**：Element Plus
- **路由**：Vue Router 4.x
- **HTTP客户端**：Axios

## 4. 开发环境配置

### 4.1 前端开发环境
1. 安装Node.js 16.x或更高版本
2. 克隆项目代码
3. 进入frontend目录
4. 安装依赖：`npm install`
5. 启动开发服务器：`npm run dev`

### 4.2 后端开发环境
1. 安装Node.js 16.x或更高版本
2. 安装MongoDB数据库
3. 克隆项目代码
4. 进入backend目录
5. 安装依赖：`npm install`
6. 配置环境变量（参考.env.example）
7. 启动开发服务器：`npm run dev`

### 4.3 管理后台开发环境
1. 安装Node.js 16.x或更高版本
2. 克隆项目代码
3. 进入admin目录
4. 安装依赖：`npm install`
5. 启动开发服务器：`npm run dev`

## 5. 核心功能模块

### 5.1 轮播图管理
- 功能：管理网站首页轮播图的展示、排序和状态
- 前端展示：使用Swiper组件实现轮播效果
- 后端API：提供完整的CRUD操作接口
- 管理后台：支持图片上传、排序调整、状态切换

### 5.2 文章管理
- 功能：管理网站的新闻、博客等文章内容
- 前端展示：文章列表和详情页面
- 后端API：支持文章的增删改查
- 管理后台：富文本编辑器、图片上传、分类管理

### 5.3 案例展示
- 功能：展示公司的成功案例
- 前端展示：案例卡片列表和详情页面
- 后端API：提供案例数据的管理接口
- 管理后台：案例信息编辑、图片上传、排序管理

### 5.4 服务介绍
- 功能：介绍公司提供的服务内容
- 前端展示：服务列表和详情页面
- 后端API：服务信息的CRUD接口
- 管理后台：服务信息编辑、图标选择、排序管理

### 5.5 内容管理
- 功能：管理网站的静态内容
- 前端展示：关于我们、联系我们等页面
- 后端API：内容的管理接口
- 管理后台：富文本编辑器、内容区块管理

## 6. API接口使用指南

### 6.1 API基础信息
- 开发环境URL：`http://localhost:3001/api`
- 生产环境URL：根据实际部署配置
- 响应格式：统一的JSON格式，包含success、data和message字段

### 6.2 主要API模块
- **轮播图API**：`/api/banners`
- **文章API**：`/api/articles`
- **案例API**：`/api/cases`
- **服务API**：`/api/services`
- **内容API**：`/api/contents`
- **文件上传API**：`/api/upload`

### 6.3 API安全考虑
- 建议在生产环境中配置API访问权限控制
- 对敏感操作添加身份验证
- 实现请求频率限制防止恶意攻击

## 7. 构建与部署

### 7.1 前端构建与部署
- 构建命令：`npm run build`
- 部署方式：Nginx静态资源服务或CDN
- 环境配置：需要配置API代理或直接指向生产环境API地址

### 7.2 后端构建与部署
- 构建命令：`npm run build`（如有TypeScript编译）
- 部署方式：PM2进程管理或Docker容器化部署
- 环境配置：数据库连接、文件上传目录、日志配置等

### 7.3 管理后台构建与部署
- 构建命令：`npm run build`
- 部署方式：Nginx静态资源服务，可与前端共享服务器
- 环境配置：API地址指向生产环境后端服务

## 8. 开发最佳实践

### 8.1 前端开发规范
- 使用TypeScript编写类型安全的代码
- 组件化开发，提高代码复用性
- 遵循Nuxt.js最佳实践，合理使用中间件、插件等特性
- 优化图片资源，实现懒加载和响应式图片
- 实现性能监控和错误追踪

### 8.2 后端开发规范
- 遵循RESTful API设计规范
- 实现统一的错误处理和日志记录
- 使用Mongoose进行数据库操作，确保数据一致性
- 实现适当的缓存机制，提高响应速度
- 定期备份数据库，确保数据安全

### 8.3 管理后台开发规范
- 遵循Vue 3 Composition API的最佳实践
- 实现统一的表单验证和状态管理
- 优化数据加载性能，实现分页和异步加载
- 提供友好的用户交互和操作反馈

## 9. 常见问题与解决方案

### 9.1 前端常见问题
- **图片加载缓慢**：优化图片尺寸、格式，使用CDN加速
- **页面渲染性能问题**：使用虚拟滚动、组件懒加载等优化手段
- **API请求失败**：实现请求重试机制和错误边界处理

### 9.2 后端常见问题
- **数据库连接失败**：实现连接重试机制和降级方案
- **文件上传失败**：检查文件大小限制和存储权限
- **服务器负载过高**：实现缓存机制和请求限流

### 9.3 管理后台常见问题
- **表单提交失败**：检查字段验证规则和API返回信息
- **数据展示错误**：验证数据格式转换和状态管理逻辑
- **权限控制问题**：检查用户权限配置和路由守卫
